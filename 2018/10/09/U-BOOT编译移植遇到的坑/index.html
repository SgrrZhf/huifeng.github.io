U-BOOT编译移植遇到的坑

发表于 2018-10-09 | 分类于 u-boot

这里记录我在学习u-boot移植过程中遇到的坑以及解决办法

u-boot1.1.6
  1. 交叉编译工具链的选择
  2. hard FP 与 software FP

u-boot1.1.6

  1. 交叉编译工具链的选择
    由于u-boot1.1.6版本的问题，不能选择eabi标准的编译器(我在编译支持nand flash新版本代码时遇到各种问题),最终选择的版本是 scratchbox-toolchain-arm-gcc4.1-uclibc20061004-1.0.4-i386.tar.gz,下载地址：http://www.scratchbox.org/download/files/sbox-releases/branches/apophis/r4/tarball/
  2. hard FP 与 software FP

由于工具链提供的libgcc.a使用的是hardfloat，u-boot1.1.6编译时给定的参数有-msoft-float，只能是在cpu/arm920t/config.mk中把这个参数去掉(这可能会导致浮点计算出问题，不确定，至少编译可以通过)
