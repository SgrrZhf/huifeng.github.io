为STM32CubeMX生成的Makefile添加下载，调试功能

发表于 2018-08-22 更新于 2019-07-26 分类于 MCU

STM32CubeMX用起来特别舒服，谁用谁知道，但是STM32CubeMX生成的Makefile只能做到编译这个功能，想要下载还得手动敲一串命令，调试也是，每次重复输入指令完全不符合程序员“懒”的属性，下面我将介绍一下我的处理方式，让make有更多属性，可以一键下载，调试，顺便也把.ycm_extra_conf.py的文件放入工程里。

需求

  1. make load命令可以直接将固件下载到MCU里
  2. make debug命令可以打开cgdb+JLinkGDBServer连接MCU调试，而且让程序直接停在main函数开头。
  3. 把编程环境配置好，让vim+ycm打开就能完美工作。（其实就是加一个.ycm…文件）

实现

make load命令下固件

这个需要配合JLinkExe来实现，JLinkExe可以执行commandfile.jlink脚本，所以事先将commandfile.jlink脚本编写好，然后往makefile里添加一条load目标，用于执行这个脚本，如下(里面的变量具体可见最后)：

make debug命令直接调试

这一条目前我还没法做到一条命令调试，需要先使用make jlinkgdbserver打开gdb调试服务，然后再使用make debug打开cgdb然后连上gdg server，运行gdbinit脚本停在main函数开头，实现需求。
