AT指令编程（数据驱动的方式）

发表于 2018-08-30 更新于 2019-07-26 分类于 coding

最近做的项目里用了一个NB-IOT的模块，用的是AT指令来控制。在这过程中，我自认为实现了一个方便、快捷的AT指令驱动，采用了数据驱动编程的思想，可以很方便的添加新指令。

前言

最早看到数据驱动编程是在2年前。当时做了一个项目，里面涉及到了一些数据传输的协议，一开始用的还是if-else、switch这种方式，程序显得特别冗长。后来看到了数据驱动编程，最后将这部分代码重构成了表驱动的方式。虽然程序的行为并没有发生改变，但是程序却变得简洁、清晰、透明了许多。

at指令基本操作方法

at指令基本就是发送命令，然后接收回复这样的操作逻辑，部分情况存在在没有发送命令的情况下直接响应的情况。具体来说一条AT指令可以用于查询，也可以用于设置，用于查询的情况时，回复的结果中就存在一些对应的数据；用于设置的情况下，发出的内容就需要附带一些有效的数据，且整个传输过程中使用的全都是字符串，很适合直接阅读。

带入数据驱动编程的思想

以AT+CPSMS（省电模式设置）为例：

用法 内容
查询 AT+CPSMS? +CPSMS:1,,,01000001,00000101
设置 AT+CPSMS=<mode> AT+CPSMS=1

对于上面CPSMS这个指令，有两种用法，查询和设置。
对于查询，使用”AT+CPSMS”，后面紧跟一个”?”,然后它的回复内容是”+CPSMS:1,,,01000001,00000101”,对于我来说，我只关系第一个参数，对其解构就是”+CPSMS:%d”,利用scanf()可以很轻松的获取到这个参数。
对于设置，发送”AT+CPSMS=<mode>”,其中<mode>可以为0,1,2。这时对其进行解构就是”AT+CPSMS=%u”,使用printf()函数可以轻易的设置这个参数。
于是整个AT指令数据模型就可以这样建立

其中名称里的AT忽略掉，回复里的OK也忽略掉。则 名称+查询格式 就是查询，用于匹配对应查询收到的数据就是查询返回格式；名称+设置格式就是设置，用于匹配对应设置收到的数据就是设置返回格式，没有就为NULL（只返回OK则视为没有）
此时可以实现一个at_read_command()用于处理查询，实现一个at_set_command()用于处理设置。
